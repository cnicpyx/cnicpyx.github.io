                                                          Dynamic Programming

                                                          Question Number 1

                                                          题意

                                                          在一条街上的房屋里抢劫，不能抢劫两间相邻的房屋，给定每个房屋里藏的钱，问：

                                                          1. 最多能抢多少？

                                                          2. 当房屋排列成一个环时，情况会发生什么改变？

                                                           

                                                          Q 1.1最大抢劫金额

                                                          算法设计

                                                          记房屋有 n 间，每间房屋里藏的钱用 v[i] 表示，其中i = 1,...,n。

                                                          从街道的左端开始遍历，记录前 i 间房屋能抢劫到的最大金额为 dp[i]。

                                                          对于第 i 间房屋，抢劫策略可以被 分为两类：

                                                          1. 抢劫该房屋

                                                          2. 不抢劫该房屋

                                                          如果选择第一种方案，那么第 i-1 间房屋一定不被抢劫，当前最大抢劫金额为：

                                                          (1)dp[i−2]+v[i]

                                                          如果选择第二种方案，那么当前最大抢劫金额应等于前 i-1 间房屋的最大抢劫金额，也就是：

                                                          (2)dp[i−1]

                                                          因此，状态转移方程应为：

                                                          (3)dp[i]=max{ dp[i−2]+v[i] dp[i−1]

                                                          初始条件为：

                                                          • dp[1] = v[1] 只有一间房屋时，肯定要选择抢劫它

                                                          • dp[2] = max(v[1], v[2]) 只有两间房屋时，要选择金额较高的抢劫

                                                          另，由于 dp[i] 的值至少与 dp[i−1] 相等，也就是说序列 dp[] 是不减的序列，因此每次状态转移时，对于“不抢劫该房屋”的情况，只要有前一项参与比较即可，而无需将前面所有项纳入比较范围。

                                                           

                                                          伪代码

                                                          Title: Q1.1 Solution
                                                          author: 庞宇轩
                                                          

                                                          function rob(v[], n)
                                                          

                                                          dp[1] = v[1];  // 只看第一间屋子，最大抢劫金额一定是该屋子的金额
                                                          dp[2] = max(v[1], v[2]);  // 只看前两间屋子，不能同时抢劫，因此要选一家金额更大的
                                                          

                                                          for i = 3 to n do
                                                              dp[i] = max(dp[i-2] + v[i], dp[i-1]);
                                                          end for
                                                          

                                                          return dp[n];

                                                          复杂度分析

                                                          伪代码中的遍历操作时间复杂度为 O(n)，其余操作均为常数，因此总时间复杂度为 O(n)。

                                                          定义一维数组 dp[] 用于储存当前最大抢劫金额，定义一维数组 v[] 用于记录每个房间藏钱的金额，二者长度均为 n，其余变量占用空间为常数，因此总的空间复杂度为 O(n)。

                                                           

                                                          Q 1.2 房屋成环

                                                          算法设计

                                                          当房屋成环时，也就是说第 1 间房屋和第 n 间房屋相邻，此时按照 是否抢劫第一间房屋 将情况分为两类：

                                                          1. 抢劫第 1 间房屋

                                                          2. 不抢劫第 1 间房屋

                                                          沿用上一小问对于变量名的定义，对于情况1，有：

                                                          • dp[1] = v[1] 因为第一间房屋被抢劫

                                                          • dp[2] = dp[1] 因为已经确定第一间房屋被抢劫，那么无论第二间房屋价值几何，都必然不会被抢劫

                                                          • dp[n] = dp[n-1] 因为第一间房屋已知被抢劫，那么与其相邻的第 n 间房屋不能被抢劫

                                                          对于情况2，有：

                                                          • dp[1] = 0 因为第一间房屋没有被抢劫

                                                          • dp[2] = v[2] 因为已经确定第一间房屋没有被抢劫，由于每间房屋储存的钱不是负数，那么第二间房屋必然要被抢劫

                                                          • 第 n 间房屋的抢劫状态不受限，可以按照状态转移方程正常判断

                                                          因此，我们采用不同的初始值，按照上面的状态转移方程，分别迭代到 dp[n-1] 和 dp[n]，选择二者较大的一项，即为这种情况下的最大抢劫金额。

                                                           

                                                          伪代码

                                                          Title: Q1.2 Solution
                                                          author: 庞宇轩
                                                          

                                                          function rob(v[], n)
                                                          

                                                          dp_1[1] = v[1];  // 第一种方案，抢劫第一间屋子
                                                          dp_1[2] = v[1];  // 由于第一间屋子确定要被抢劫，因此第二间屋子不能被抢劫
                                                          

                                                          for i = 3 to n-1 do  // 迭代到倒数第二间屋子
                                                              dp_1[i] = max(dp_1[i-2] + v[i], dp_1[i-1]);
                                                          end for
                                                          

                                                          dp_2[1] = 0;  // 第二种方案，第一间屋子不抢
                                                          dp_2[2] = v[2];  // 当只有两间屋子时，第一间已经确定不抢，第二间一定要抢
                                                          

                                                          for i = 3 to n do  // 迭代到最后一间屋子
                                                              dp_2[i] = max(dp_2[i-2] + v[i], dp_2[i-1]);
                                                          end for
                                                          

                                                          return max(dp_1[n-1], dp_2[n]);  // 返回二者较大值

                                                           

                                                          复杂度分析

                                                          本算法新增了一轮遍历，但两轮遍历的时间复杂度均为 O(n)，因此总的时间复杂度仍为 O(n)。

                                                          类似地，本算法新增了一个一维数组，长度为 O(n)，因此总的空间复杂度仍为 O(n)。

                                                          为节省空间，可以将两轮数据都储存在同一 dp[] 中，只需记录第一轮迭代的结果即可。

                                                           

                                                          Question Number 2

                                                          题意

                                                          定义“丑数”为 质因子只有 2, 3, 5 的正整数，给定一个正整数 n 求出第 n 个丑数。

                                                          Q 2.1 暴力法

                                                          算法设计

                                                          首先，最小前几个丑数依次为 1, 2, 3, 4=2x2, 5, 6=2x3, 8=4x2，可以看出，丑数都是 由现有丑数乘以 2 或 3 或 5 得出。

                                                          因此，我们可以维护一个丑数列表，一边 找当前最小的丑数，一边 用当前最小丑数乘以 2、3、5 来向列表中新增丑数，直到找到第 n 个丑数为止。

                                                          具体来说，维护一个列表 list，将三个初始值 2,3,5 加入列表，此后：

                                                          • 遍历列表，找到最小值 x，将其推出列表（可以通过将其标记为已访问实现）

                                                          • 将 2x, 3x, 5x 加入列表

                                                          重复上述步骤，直到找到第 n-1 个最小值，即为总的第 n 个“丑数”。

                                                           

                                                          伪代码

                                                          Title: Q2.1 Solution
                                                          author: 庞宇轩
                                                          

                                                          function find_ugly_number_1(n)
                                                          

                                                          let nth_ugly_number = 1;  // 用于储存结果
                                                          

                                                          let already_find = 1;  // 记录已经找到的丑数个数
                                                          let ugly_list = {2, 3, 5};  // 储存当前所有的丑数
                                                          

                                                          while already_find < n do  // 没找够
                                                              // 找到当前最小丑数
                                                              let cur_min = 0;
                                                              for num in ugly_list do
                                                                  cur_min = min(cur_min, num);
                                                              end for
                                                              already_find ++;  // 找到丑数的个数 + 1
                                                              ugly_list.pop(cur_min);  // 当前丑数推出数组
                                                              if already_find == n then  // 找到第 n 个丑数，记录
                                                                  nth_ugly_number = cur_min;
                                                                  break;
                                                              end if
                                                              
                                                              // 新增丑数
                                                              ugly_list.push(2 * cur_min);
                                                              ugly_list.push(3 * cur_min);
                                                              ugly_list.push(5 * cur_min);
                                                              
                                                          end while
                                                          

                                                          return nth_ugly_number;

                                                           

                                                          证明

                                                          由于本题在算法设计过程中的推理逻辑并不严密，特给出算法正确性的证明。

                                                          首先，我们定义，对于两个丑数 a 和 b：

                                                          • 如果满足 b≡0moda，则称 a 是 b 的子数；

                                                          • 如果满足 a∗2=b 或 a∗3=b 或 a∗5=b，则称 a 是 b 的最近子数，b 是 a 的最近父数；

                                                          如果存在一个丑数 x 属于前 n 个丑数，但未被找到，那么在我们找到的 n 个丑数中，必然存在一个丑数 y>x。

                                                          由于被选中的丑数均为当前列表 ugly_list 中的最最小值，如果 x 未被找到而更大的 被找到了，说明 x 必然从未进入列表 ugly_list。

                                                          由于列表元素增加的唯一途径是 将选中元素的最近父数存入列表，如果 x 从未进入列表，那么 x 的最近子数 x′<x 必然从未被选中。

                                                          由于 y>x>x′ 被选中，而更小的 从未被选中，说明 从未进入列表。

                                                          经过上面的推理，可知：对于一个从未进入列表的丑数，如果存在一个比它大的丑数被选中，那么它的最近子数必然从未进入列表。

                                                          所有丑数的素因子有且仅有 2,3,5，而递归地重复上述结论，得出结论 2,3,5 中必然有至少一个元素从未进入列表，这与事实相违背。

                                                          因此，假设不成立，反证可知这样的 n 个丑数不存在遗漏。

                                                           

                                                          复杂度分析

                                                          易知，外层循环共执行 n 次，内层循环的执行次数即为 当前丑数列表的长度。

                                                          丑数列表初始长度为 3，外层循环每执行一次，有：

                                                          • 一个元素被推出列表

                                                          • 三个元素被压入列表

                                                          因此，可以得到总的运算次数为：

                                                          (4)3+(3+2)+(3+2+2)+⋯+(2n+1)=n(n+2)

                                                          其他操作的时间均为常数，因此总的时间复杂度为 O(n2)。

                                                          一维数组 ugly_number 占用空间最大为 ，其余变量占用空间为常数，因此总的空间复杂度为 O(n)。

                                                          上述复杂度分析中，忽略了“推出数组”操作所占用的时间。若采用 “标记为已访问” 来实现 “推出数组” 的效果，那么丑数列表长度不会减少，最终长度应为 3n，另需设置一个数组 vis[] 用于记录被访问过的点。按照这样的方法，时间和空间复杂度会小幅度增加，但仍保持 O(n2) 和 O(n) 的数量级。

                                                           

                                                          Q 2.2 动态规划法

                                                          算法设计

                                                          参考暴力法的思路可以得知，后续的每个丑数都是 由前面某一个丑数乘 2 或 3 或 5 得来的。

                                                          记当前丑数为 dp[i]，它的来源有三种可能：

                                                          (5)dp[i]=min{ dp[index2]∗2 dp[index3]∗3 dp[index5]∗5

                                                          其中，我们从起点开始，维护三个下标 index_2, index_3, index_5，每次选择哪一项，哪一个下标就 +1。

                                                          重复上述过程，直到找到第 n 个丑数为止。

                                                          这样的状态转移方程通过每轮取最小值，模拟了三个因子逐步增加的过程，因此必然可以没有遗漏地找到前 n 个丑数。

                                                           

                                                          伪代码

                                                          Title: Q2.2 Solution
                                                          author: 庞宇轩
                                                          

                                                          function find_ugly_number_1(n)
                                                          

                                                          let ugly_list = {1};
                                                          let index_2 = index_3 = index_5 = 0;  // 数组下标从 0 开始
                                                          

                                                          for i = 1 to n-1 do
                                                              let cur_min = ugly_list[index_2] * 2;
                                                              cur_min = min(cur_min, ugly_list[index_3] * 3);
                                                              cur_min = min(cur_min, ugly_list[index_5] * 5);
                                                              
                                                              ugly_list.push(cur_min);  // 新增的丑数入队
                                                              if cur_min == ugly_list[index_2] * 2 then
                                                                  index_2 ++;
                                                              end if
                                                              if cur_min == ugly_list[index_3] * 3 then
                                                                  index_3 ++;
                                                              end if
                                                              if cur_min == ugly_list[index_5] * 5 then
                                                                  index_5 ++;
                                                              end if
                                                              
                                                          end for
                                                          

                                                          return ugly_list[n-1];

                                                           

                                                          复杂度分析

                                                          算法只包含一层循环，该循环执行次数为 n-1，其余操作均为常数，因此总的时间复杂度应为 O(n)。

                                                          一维数组 ugly_number 占用空间最大为 n，其余变量占用空间为常数，因此总的空间复杂度为 O(n)。

                                                           

                                                          Question Number 3

                                                          题意

                                                          给定一个正整数 n，求由序列 1...n 组成的二叉搜索树有多少种不同的结构。

                                                          二叉搜索树：

                                                          • 左子树上的所有节点小于根节点

                                                          • 右子树上的所有节点大于根节点

                                                          • 左子树和右子树均为二叉搜索树

                                                          算法设计

                                                          从二叉搜索树的定义中可以看出，左子树和右子树均为二叉搜索树，可以由这一递归关系设计状态转移方程：

                                                          定义对于正整数 n，有 dp[n] 种不同的二叉搜索树，这些二叉搜索树的根节点有 n 种取值：1,...,n。

                                                          对于任意一种根节点 r：

                                                          • 它的左子树一定是一棵节点数为 r-1 的二叉搜索树，这样的二叉搜索树有 dp[r-1] 棵

                                                          • 它的右子树一定是一棵节点数为 n-r 的二叉搜索树，这样的二叉搜索树有 dp[n-r] 棵

                                                          因此，有状态转移方程：

                                                          (6)dp[i]=∑r=1idp[r−1]∗dp[i−r]

                                                          其初始值为：

                                                          • dp[0] = 0 （不计空树）

                                                          • dp[1] = 1

                                                          • dp[2] = 2

                                                          • dp[3] = 5

                                                           

                                                          伪代码

                                                          Title: Q3 Solution
                                                          author: 庞宇轩
                                                          

                                                          

                                                          function unique_BSTs(n)
                                                          

                                                          dp[] = {0,1,2,5};  // 初始化数组
                                                          

                                                          for i in 4 to n do  // 当前节点总数
                                                              bsts = 0;
                                                              for r in 1 to i do  // 共有i个节点时，根节点为r的BST数目
                                                                  bsts += dp[r-1] * dp[i - r];
                                                              end for
                                                              dp.push(bsts);  // 加入数组
                                                          end for
                                                          

                                                          return dp[n];

                                                           

                                                          复杂度分析

                                                          外层循环执行 (n−4) 次，内层循环执行次数与当前节点总数有关，总执行次数为：

                                                          (7)4+5+⋯+n=(n+4)(n−4)2

                                                          其他操作时间复杂度均为常数，因此总时间复杂度为 O(n2)。

                                                          一维数组 dp[] 长度为 n，其余变量占用空间均为常数，因此总空间复杂度为 O(n)。

                                                           

                                                           

                                                          Question Number 4

                                                          题意

                                                          给定一组不同的正整数，找出最大的子集，满足：对于任意的整数对 (si,sj)，均有 si%sj=0 或 sj%si=0。

                                                          算法设计

                                                          由于这一组正整数互不相同，那么对于满足 si%sj=0 或 sj%si=0 的整数对，一定是 一大一小。

                                                          受此启发，我们如果 从小向大找，那么新增元素的充分必要条件是 新增的数一定可以整除现有子集中的所有数。

                                                          又因为现有子集中的最大值一定可以整除所有数，因此这一条件可以等价为 新增的数一定可以整除现有子集中的最大值。

                                                          综上，对于一个严格递增的序列 nums[]，当其中的某一元素 nums[i] 充当子集中的最大元素时，子集的最大元素个数记为 dp[i]，有

                                                          (8)dp[i]=maxj{ dp[j]+1,nums[i]≡0modnums[j], i>j 1

                                                          初始值为：

                                                          • dp[0] = 1

                                                           

                                                          伪代码

                                                          xxxxxxxxxx
                                                          Title: Q4 Solution
                                                          author: 庞宇轩
                                                          

                                                          function Latgest_Divide_Subset(nums[])
                                                          

                                                          let n = nums.size()  // 记录数组长度
                                                          sort(nums.begin(), nums.end());  // 升序排序原数组
                                                          

                                                          dp[n] = {1};
                                                          

                                                          for i in 1 to n-1 do  // 从小到大遍历，每个新访问的元素都严格大于之前所有元素
                                                              dp[i] = 1;  // 该元素本身必然属于子集
                                                              
                                                              for j in 0 to i-1 do
                                                                  if nums[i] % nums[j] = 0 then  // 找到能被nums[i]整除的nums[j]
                                                                      dp[i] = max(dp[i], dp[j] + 1);  // dp[j]的所有元素均可加入dp[i]的子集
                                                                  end if
                                                              end for
                                                              
                                                          end for
                                                          

                                                          let max_size = 1;
                                                          

                                                          for i in 0 to n-1 do
                                                              max_size = max(max_size, dp[i]);
                                                          end for
                                                          

                                                          return max_size;

                                                           

                                                          复杂度分析

                                                          二层循环的外层循环执行 (n−1) 次，内层循环执行次数递增，总执行次数为：

                                                          (9)1+⋯+n−1=n(n−1)2

                                                          即二层循环的时间复杂度为 O(n2)，又有排序的时间复杂度为 O(nlog⁡n)，后续寻找最大值的循环时间复杂度 O(n)，因此总的时间复杂度应为 O(n2)。

                                                          一维数组 dp[] 和 nums[] 占用空间均为 O(n)，其余变量占用空间为常数，因此总的空间复杂度应为 O(n)。

                                                           

                                                          Question Number 5

                                                          题意

                                                          给定一个整数序列和一个目标数 target，为序列中的每个整数前面添加符号，组成一个计算式，这个计算式的值恰好为 target，求这样的计算式的个数。

                                                          算法设计

                                                          遍历给定的整数序列 arr[]，对于当前的元素 arr[i]，前面的符号有两种选择：+arr[i] 和 -arr[i]。

                                                          这两种选择会导致当前的算式指向两个不同的值，这样的“值”的范围处于 [−sum,sum] 之间，其中 sum 为序列所有元素绝对值的和。

                                                          定义一个二维数组 dp[i][j]，用于记录 arr[] 中下标不超过 i 的元素组合出的算式，指向值 j 的次数，有状态转移方程：

                                                          (10)dp[i][j]=∑jdp[i−1][j±arr[i]]

                                                           

                                                          例如，在本题给出的示例中，arr[] = {1,1,1,1,1}, target=3，dp[][] 为：

                                                          可以看出，dp[4][3] 即为所求解。

                                                          由于下标不能出现负数，因此需要将算式的值平移至原点右侧，通过 +sum 实现。

                                                          伪代码

                                                          Title: Q5 Solution
                                                          author: 庞宇轩
                                                          

                                                          function Target_Sum(arr[], target)
                                                          

                                                          let sum = 0;
                                                          for a in arr do
                                                              sum += abs(a);
                                                          end for
                                                          

                                                          创建二维数组dp[][] 第一维长度等于arr[]长度，第二维长度等于2sum+1，值初始化为0
                                                          

                                                          // 为 dp 赋初始值
                                                          // 只有一个元素 arr[0] 时，算式只有两种值：+arr[0] 和 -arr[0]，各出现一次
                                                          // 由于 arr[0] 可能是 0，因此要采用 += 来赋值，而不能直接使用 =
                                                          dp[0][sum - arr[0] ] += 1;
                                                          dp[0][sum + arr[0] ] += 1;
                                                          

                                                          for i in 1 to arr.size()-1 do
                                                              for j in 0 to 2sum+1 do
                                                                  if dp[i-1][j] != 0 then  // 前面的某些算式到达过值 j
                                                                      dp[i][j + arr[i]] += dp[i-1][j];  // 新算式可以借此到达值 j+arr[i]
                                                                      dp[i][j - arr[i]] += dp[i-1][j];  // 新算式可以借此到达值 j-arr[i]
                                                                  end if
                                                              end for
                                                          end for
                                                          

                                                          return dp[arr.size()-1][sum + target];

                                                           

                                                          复杂度分析

                                                          外层循环执行次数 (n−1)，内层循环每次执行 (2sum−1) 次，即二层循环共执行 (n−1)(2sum−1) 次，时间复杂度为 O(n×sum)

                                                          定义二维数组 dp 占用空间 n×(2sum−1)，数组 arr[] 占用空间 O(n)，因此总的空间复杂度应为 O(n×sum)。

                                                          可以使用两个长度为 (2sum−1) 的一维数组来代替二维数组 dp，此时要根据 n 和 (2sum−1) 的相对大小来判断空间复杂度为 O(n) 或 O(sum)
