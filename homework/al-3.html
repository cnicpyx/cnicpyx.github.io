                              Question Number 1

                              题意

                              有 n 个任务，需要安排给 n 个士兵来完成，对于每个任务 i，需要花费 bi 的时间给士兵布置任务，布置完成后，士兵需要花费 ji 的时间来完成任务。

                              士兵完成任务是相对独立的，也就是说它们完成任务的时间段可以重复。

                              求从布置第一个任务开始，到所有任务均被完成，所需要的最短时间。

                              算法设计

                              首先，无论采用何种顺序，布置任务消耗的总时长均为：

                              b1+b2+⋯+bn=sumb

                              对于其中的第 k 个任务，布置该任务花费时间 bk，士兵完成任务花费时间 jk，因此，从开始布置第一个任务到任务 k 完成，所花费的时间为：

                              Tk=b1+⋯+bk−1+bk+jk

                              当且仅当 Tk>sumb 时，也就是在布置完所有任务之后，任务 k 被完成，才会产生额外的时间。对于满足条件的所有 k，取其中最大的 Tk，即为最终完成任务所需的时间，我们的目标是让这个最大的 Tk 最小。

                              可以表达为：

                              mink{ maxTk>sumb{∑i=1kbk+jk−sumb} }

                              可以看出，要使最大的 Tk 最小，须将 更大的 jk 分配给更小的 k。

                              因此，贪心的策略为 按士兵完成任务的时间降序排序，计算每一个 Tk 并取最大值即可。

                              Question Number 2

                              题意

                              给定若干个长度相等的字符序列，找到它们的 一致性字符串 并计算 共识误差，其中：

                              • 一致性字符串：与其他所有字符串的汉明距离最小的字符串

                              • 共识误差：该汉明距离的值

                               

                              算法设计

                              依次计算字符串每一位的词频：

                              • 选择词频最高的字母，作为 一致性字符串该位置的字母（若有多个满足条件则取字典序最低）

                              • 将其他所有词频求和，作为 该位置的共识误差，最终对所有位的共识误差求和即可

                              Question Number 3

                              题意

                              埃利亚与 n 个人打架，当且仅当 n 个人全部到场时埃利亚失败，否则埃利亚胜利。给定这 n 个人在 d 天中的出勤情况表，求埃利亚最大的连胜天数。

                               

                              算法设计

                              根据出勤情况表，计算这 d 天里，每天是否全员到场。定义一个 bool 型数组 all 记录这一计算结果，all[i] = true 表示第 i 天所有人都来了，反之则表明有人没来。

                              遍历序列 all[]，求最大的连续 0 的个数即可。

                              • 首先要保证 位数尽可能的小，也就是说将数字 s 拆成更大的个位数

                              • 其次要保证 更大的数位于更低的位，也就是说应从个位开始往回依次放置 9 8 ...
